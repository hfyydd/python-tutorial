Python Object-Oriented Programming

Object-Oriented Programming (OOP) is a programming paradigm that uses "objects" to design software. Python is an object-oriented language.

Basic OOP Concepts

üí° Core OOP Concepts:
  • Class: A blueprint or template for creating objects
  • Object: An instance of a class
  • Attribute: Data associated with an object
  • Method: Behavior associated with an object

Defining Classes

Use the class keyword to define a class:

class Dog: """A class representing a dog""" # Class attribute species = "Mammal" # Initialization method (constructor) def __init__(self, name, age): self.name = name # Instance attribute self.age = age # Instance method def bark(self): print(f"{self.name} says Woof!") def describe(self): print(f"{self.name} is {self.age} years old")

Creating Objects

# Create objects (instantiation) dog1 = Dog("Buddy", 3) dog2 = Dog("Max", 2) # Call methods dog1.bark() # Buddy says Woof! dog1.describe() # Buddy is 3 years old # Access attributes print(dog2.name) # Max print(dog2.species) # Mammal

The self Parameter

self represents the object itself. It must be the first parameter of class methods:

class Person: def __init__(self, name): self.name = name def say_hello(self): print(f"Hello, I'm {self.name}") person = Person("Alice") person.say_hello() # Hello, I'm Alice

Class Methods and Static Methods

class MathUtils: # Class method @classmethod def get_pi(cls): return 3.14159 # Static method @staticmethod def add(a, b): return a + b # Call class method print(MathUtils.get_pi()) # 3.14159 # Call static method print(MathUtils.add(5, 3)) # 8

Inheritance

Inheritance allows us to create a new class that inherits attributes and methods from an existing class:

class Animal: def __init__(self, name): self.name = name def eat(self): print(f"{self.name} is eating") # Dog class inherits from Animal class Dog(Animal): def bark(self): print(f"{self.name} says Woof!") # Create Dog object dog = Dog("Buddy") dog.eat() # Inherited method: Buddy is eating dog.bark() # Own method: Buddy says Woof!

Method Overriding

class Animal: def speak(self): print("Animal makes a sound") class Cat(Animal): # Override parent method def speak(self): print("Meow") cat = Cat() cat.speak() # Meow

Encapsulation

Use underscores to indicate private attributes and methods:

class BankAccount: def __init__(self, balance): self.__balance = balance # Private attribute # Provide methods to access private attributes def get_balance(self): return self.__balance def deposit(self, amount): if amount > 0: self.__balance += amount account = BankAccount(1000) print(account.get_balance()) # 1000 account.deposit(500) print(account.get_balance()) # 1500

Polymorphism

class Shape: def area(self): pass class Rectangle(Shape): def __init__(self, width, height): self.width = width self.height = height def area(self): return self.width * self.height class Circle(Shape): def __init__(self, radius): self.radius = radius def area(self): return 3.14159 * self.radius ** 2 # Polymorphism: different objects call same method shapes = [Rectangle(5, 3), Circle(2)] for shape in shapes: print(shape.area())
